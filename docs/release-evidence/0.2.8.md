## Automated Evidence

- Release prep:
  - Package version: `0.2.8`
  - Planned tag: `v0.2.8`
  - Publish path: tag-triggered GitHub Actions trusted publishing
- Release scope since `0.2.7`:
  - Stable live release sign-off now gates npm `latest`, GitHub latest, and the
    GitHub Release `install.sh` asset only.
  - npm `next` is treated as an explicit prerelease-only channel. Stable release
    completion no longer requires `next >= latest` or a maintainer
    `npm dist-tag add ... next` mutation.
  - `releaseIntegrity.npm.nextDistTagAction` remains in live JSON output for
    compatibility and reports `not_applicable` during stable sign-off.
  - The local publish helper verifies `latest` only for stable fallback
    publishes; `publish:next` remains the explicit prerelease path.

## Manual E2E Sign-Off

No real-model E2E is required before tag publish for `0.2.8`. The release is a
release-channel policy and release-integrity hardening slice. It does not add
live model execution, app health authority, cleanup authority, prompt
injection, or new completion authority.

## Deferred Surfaces

The following remain outside the stable contract:

- live `cx autopilot run`;
- unattended cron/gateway scheduler ownership, leases, and retry;
- LSP protocol-server lifecycle and automatic project LSP application;
- Desktop app-server live attachment;
- automatic context or prompt injection;
- app health authority and automatic cleanup.
