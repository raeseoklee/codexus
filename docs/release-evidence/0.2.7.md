stdout dumps, tokens, local paths that reveal private projects, or model prompts.
Record only bounded evidence needed to prove the release checklist.

## Automated Evidence

- Release prep:
  - Package version: `0.2.7`
  - Planned tag: `v0.2.7`
  - Publish path: tag-triggered GitHub Actions trusted publishing
- Release scope since `0.2.6`:
  - `release check --gate` now reports
    `releaseIntegrity.installScript.helpNonDestructive:true` only when
    `install.sh` parses `-h`/`--help` and unknown options before Node/npm
    probes, `npm install`, bin-link mutation, or Codex skill adapter
    installation.
  - A regression test proves that an installer without non-destructive help
    handling fails the release gate before publish.
  - Documentation now lists installer help safety as part of local release
    integrity and the release checklist.

## Manual E2E Sign-Off

No real-model E2E is required before tag publish for `0.2.7`. The release is a
release-integrity hardening slice and does not add live model execution, app
health authority, cleanup authority, prompt injection, or new completion
authority.

## Deferred Surfaces

The following remain outside the stable contract:

- live `cx autopilot run`;
- unattended cron/gateway scheduler ownership, leases, and retry;
- LSP protocol-server lifecycle and automatic project LSP application;
- Desktop app-server live attachment;
- automatic context or prompt injection;
- app health authority and automatic cleanup.
