Worktree App Instance Launcher

worktree evidence experimental

Codexus does not yet boot one application instance per git worktree. Add a descriptor-backed launcher before browser, log, or app-health evidence can be tied to per-change instances.

Acceptance
  • cx app instance start/status/stop/logs --json records an owned instance artifact.
  • Each artifact includes worktree path, branch/head, command profile, pid/owner heartbeat, port or URL, health check, log paths, and status.
  • Port allocation is per worktree; stop cleans only the owned process and never unrelated local servers.
  • Browser/dev-server evidence references an instance id and never claims control or health without live evidence.

3. Gated / Blocked

Known valuable features that must stay visibly unavailable until the upstream or local contract exists.

blocked experimental

Do not enable live turn execution until Desktop observation and session-event mapping are proven separately.

Gate

Supported daemon observation + non-disruptive attach + event mapping + separate enablement design.

Real Cron / Gateway Dispatch

blocked automation

Current surfaces may report status and dry-run audit records only.

Gate

Permission, approval, lock, dispatch, completion, and recovery events must be live.

Statusline Integration

blocked upstream

Keep cx session hud --json as the supported fallback until Codex exposes a stable statusline configuration surface.

Gate

Supported Codex configuration surface with documented lifecycle and failure behavior.

tmux-Backed Worker Launch

blocked launch contract

session workers status should remain a gate report until session state and launch protocol are stable.

Gate

Stable worker protocol, ownership model, cancellation behavior, and evidence separation.

Native Subagent Active Launcher

blocked bridge required

Keep record/attach/complete/launch-contract surfaces, but do not claim Codexus can spawn native subagents yet.

Gate

A supported Codex bridge plus verification-only completion policy.

Automatic Context Injection

blocked approval required

Retrieved context stays as visible approval artifacts until a reversible injection path is designed.

Gate

User-visible approval, audit artifact, disable path, and no silent prompt mutation.
