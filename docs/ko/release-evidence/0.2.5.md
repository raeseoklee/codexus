# 0.2.5 Release Evidence

[English](../../release-evidence/0.2.5.md)

Status: `0.2.5` prepared; post-publish live sign-off pending

이 파일은 evidence operations pack을 위한 redacted release-prep evidence
artifact입니다. Raw run ledger, transcript, stdout dump, token, private project
local path, model prompt를 커밋하지 마세요. Release checklist를 증명하는 데 필요한
bounded evidence만 기록합니다.

## Automated Evidence

- Release prep:
  - Package version: `0.2.5`
  - Planned tag: `v0.2.5`
  - Publish path: tag-triggered GitHub Actions trusted publishing
- `0.2.4` 이후 release scope:
  - `cx app instance profile doctor --json`은 앱을 시작하거나 제어하지 않고
    app-instance descriptor preflight evidence를 read-only로 보고합니다.
  - `cx app instance observe --json`은 process, heartbeat, port, HTTP projection,
    log signal을 분리해 보고하지만 health, cleanup, control, completion authority가
    되지 않습니다.
  - `cx app instance evidence collect --json`은 하나의 `instanceId`에 대해 bounded
    metrics, logs, loopback HTTP evidence pack을 기록합니다.
  - `cx evidence status --json`은 app-instance evidence, deterministic wiki status,
    LSP detection status를 집계하는 experimental project evidence dashboard를
    제공합니다.
  - `cx evidence check --gate --json`은 기존 repo, wiki, LSP, release gate를
    집계하지만 semantic completion authority가 되지 않습니다.
  - `cx evidence export --target <path> --json`은 사람이 검토할 수 있는
    markdown/JSON projection을 명시적으로 쓰며, symlink된 workspace-local target을
    거부하고 file-byte hash를 보고하며 auto-commit하지 않습니다.
  - Feature reference, implementation status, remaining-work backlog, roadmap
    kanban, project wiki current-state, changelog, release evidence 문서를 tag publish
    전에 갱신했습니다.

## Manual E2E Sign-Off

`0.2.5` tag publish 전 real-model E2E는 요구하지 않습니다. 이 release는 bounded
evidence operations와 app-instance observation helper만 추가합니다. Live model
execution, automatic installation mutation, automatic prompt injection, app health
authority, cleanup authority, 새로운 completion authority는 추가하지 않습니다.

## Deferred Surfaces

다음 항목은 stable contract 밖에 남아 있습니다:

- live `cx autopilot run`;
- unattended cron/gateway scheduler ownership, lease, retry;
- LSP protocol-server lifecycle과 automatic project LSP application;
- Desktop app-server live attachment;
- automatic context 또는 prompt injection;
- app health authority와 automatic cleanup.
