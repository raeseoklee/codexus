## 자동 증거

- Release prep:
  - Package version: `0.1.7`
  - Planned tag: `v0.1.7`
  - Publish path: tag-triggered GitHub Actions trusted publishing
- `0.1.6` 이후 release scope:
  - Experimental project LSP diagnostics evidence:
    `cx lsp status`는 language server를 시작하지 않고 project diagnostic candidate를
    탐지하며, `cx lsp check [--gate]`는 `npm run typecheck` 같은 명시적 project
    diagnostics를 실행합니다.
  - `npm run lsp:check`는 `npm run release:check`에 포함되고, package smoke는 설치된
    CLI가 `cx lsp check --gate`를 실행할 수 있음을 검증합니다.
  - Update availability notification은 이 release에서 구현되지 않았습니다. 문서화된 첫
    slice는 informational, TTL-cached, opt-out-capable이어야 하며 primary command를
    실패시키거나 설치를 변경하면 안 됩니다.
  - Codex plugin packaging은 experimental distribution/discoverability layer로만
    계획됐습니다. npm-installed `$codexus` skill은 stable Codex-native adapter로
    유지되고, plugin 설치는 always-on supervision의 증거가 아닙니다.
  - Stable JSON contract: frozen stable field 제거/재정의 없음.
- Tag publish 전 local release gate:

    `node src/cli/main.ts release check --version 0.1.7 --gate --json`,

## 수동 E2E Sign-Off

`0.1.7` tag publish 전 새 real-model E2E는 필요하지 않습니다. 새 runtime surface는
명시적 project command 위의 experimental LSP diagnostics evidence뿐입니다. Protocol
server를 시작하지 않고, 파일을 수정하지 않으며, completion authority를 주장하지
않습니다. Update notification과 plugin packaging은 이 release에서 planning-boundary
documentation입니다.
