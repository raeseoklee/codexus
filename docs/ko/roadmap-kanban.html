Codexus 로드맵 칸반

0.1.6 release 이후 indexed Codexus roadmap 작업을 4컬럼으로 정리한 문서입니다. 각 카드는 Codexus의 evidence-first 원칙에 맞춰 작성했습니다. 로컬 fact는 gate할 수 있고, 판단이 필요한 항목은 advisory로 남기며, runtime contract가 없는 기능은 명시적으로 blocked 상태로 둡니다.

갱신일: 2026-06-04 기준: npm 0.1.6 published 출처: indexed docs 01-19 + remaining work English 남은 작업 하네스 정렬 문서

1. 완료 / 바로 가능

최근 완료된 evidence 작업과 deferred runtime을 켜지 않고 구현할 수 있는 작은 derivable gate입니다.

triage contract safe

완료 조건
  • Stable JSON field는 additive-only입니다.
  • Experimental surface는 stable처럼 보이지 않습니다.

Instance-Linked App Evidence

worktree evidence experimental HTTP/log 완료

Live ownership, observation descriptor, stale/orphan lifecycle policy baseline, loopback HTTP dev-server probe, bounded log snapshot adapter는 구현됐습니다. 다음 단계는 더 풍부한 Browser/DevTools/screenshot/metric adapter입니다.

완료 조건
  • 구현된 baseline은 정직해야 합니다. Codexus-owned instance만 중지할 수 있고, stale/orphan artifact는 lifecycle policy를 보고하되 health, cleanup, completion, control을 주장하지 않습니다.
  • 완료: loopback HTTP probe evidence는 instanceId를 참조하고 bounded/redacted이며 control, health, completion authority를 주장하지 않습니다.
  • 완료: log snapshot evidence는 같은 instanceId에 대해 bounded/redacted stdout/stderr tail을 기록하되 health, control, completion authority가 되지 않습니다.
  • 다음: Browser/DevTools/screenshot/metric 출력은 “이 server를 관측했다”와 “이 owned worktree instance를 관측했다”를 구분할 수 있어야 합니다.
  • Future autopilot surface는 generic localhost port가 아니라 per-worktree instance artifact를 재사용해야 합니다.

3. Gated / Blocked

가치는 있지만 upstream 또는 local contract가 생길 때까지 unavailable로 보여야 하는 기능입니다.

Live App-Server Turn Execution

blocked experimental

Desktop 관측과 session-event mapping이 별도로 증명되기 전까지 live turn execution을 켜지 않습니다.

Gate

지원되는 daemon 관측, non-disruptive attach, event mapping, 별도 enablement 설계.

Automation Scheduler / Recovery Ownership

experimental live slice automation

Explicit-approval live dispatcher와 blocked-dispatch boundary audit record는 이미 있습니다. 남은 작업은 autonomy를 과장하지 않으면서 unattended scheduler, retry, ownership evidence를 추가하는 것입니다.

Gate

Permission, approval, lock, dispatch, completion, recovery, owner-liveness evidence가 durable해야 합니다.

Statusline Integration

blocked upstream

Codex가 안정적인 statusline configuration surface를 제공할 때까지 cx session hud --json을 fallback으로 유지합니다.

Gate

문서화된 lifecycle/failure behavior를 가진 Codex 지원 설정면.

tmux-backed Worker Launch

blocked launch contract

Session state와 launch protocol이 안정될 때까지 session workers status는 gate report로 유지합니다.

Gate

Worker protocol, ownership model, cancellation behavior, evidence separation.

Native Subagent Active Launcher

blocked bridge 필요

Record/attach/complete/launch-contract surface는 유지하지만, Codexus가 native subagent를 spawn한다고 주장하지 않습니다.

Gate

지원되는 Codex bridge와 verification-only completion policy.

Automatic Context Injection

blocked approval 필요

Retrieved context는 reversible injection path가 설계되기 전까지 visible approval artifact로만 둡니다.

Gate

User-visible approval, audit artifact, disable path, no silent prompt mutation.

Plugin Install-State Diagnostics

experimental packaging 구현됨 install state deferred

Packaged plugin과 cx plugin status는 이제 manifest freshness를 검증합니다. 다음은 Codex가 문서화된 plugin install-location contract를 제공한 뒤 installed plugin state를 진단하는 것입니다.

Gate

Plugin 설치가 always-on supervision을 암시하면 안 됩니다. 관측된 heartbeat나 notify evidence가 여전히 권한입니다.
