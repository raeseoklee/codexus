Codexus 로드맵 칸반

0.1.2 release-candidate 기준, indexed 문서에 계획되어 있지만 아직 구현되지 않은 Codexus 작업을 4컬럼으로 정리한 문서입니다. 각 카드는 Codexus의 evidence-first 원칙에 맞춰 작성했습니다. 로컬 fact는 gate할 수 있고, 판단이 필요한 항목은 advisory로 남기며, runtime contract가 없는 기능은 명시적으로 blocked 상태로 둡니다.

1. 바로 가능

Deferred runtime을 켜지 않고도 구현할 수 있는 작고 도출 가능한 유지보수/gate입니다.

0.1.x Patch Queue

triage contract safe

작은 0.1.x 작업을 모아두는 holding lane입니다. Stable surface에는 additive만 적용하고, 새 runtime surface는 experimental/deferred로 자기보고해야 합니다.

완료 조건
  • Stable JSON field는 additive-only입니다.
  • Experimental surface는 stable처럼 보이지 않습니다.
  • Gate 가능한 claim은 derivable fact로 뒷받침됩니다.

Control-Plane Follow-Up

17번 문서 advisory-first

Decision artifact, loop summary, HUD projection은 구현됐습니다. 다음은 새 완료 권한을 만들지 않고 control-plane metadata를 확장하는 작업입니다.

완료 조건
  • Autonomy preset name은 trust score가 아니라 contract metadata입니다.
  • Policy catalog output은 enforced, observed, advisory, unavailable을 분리합니다.
  • 더 풍부한 risk fact는 derivable로 유지하고 heuristic risk는 advisory로 둡니다.
  • Task artifact는 verification evidence가 생기기 전까지 read-only projection입니다.

15번 문서 adapter 이전

Active multi-engine adapter를 승격하기 전에 deferred 상태인 acceptance-criteria-to-verification matrix를 structural relay gate로 만듭니다.

완료 조건

  • 승인된 모든 acceptance criterion은 최소 하나의 row 또는 승인된 deferred reason에 매핑됩니다.
  • 필수 matrix row에 evidence path가 없으면 implementation convergence가 실패합니다.
  • Gate가 생긴 뒤에만 현재 verification_matrix_enforcement_deferred advisory claim을 제거합니다.

2. 다음 Evidence Slice

더 큰 runtime을 켜기 전에 먼저 증거를 만들어야 하는 작업입니다.

App-Server Stdio Observer 설계

discovery 이후 remote control 금지

Discovery 결과 maintainer Desktop surface는 managed control socket이 없는 stdio-only입니다. Session-event mapping 전에 non-disruptive observer를 설계합니다.

완료 조건
  • cx app-server discover --record --json가 read-only를 유지하고 stdio-only evidence를 기록합니다.
  • Observer 설계는 기존 stdio pipe에 destructive하게 붙지 않습니다.
  • Positive mapping은 transcript value 없이 관측된 turn-boundary evidence가 있을 때만 가능합니다.

Release Integrity Live Sign-Off

post-publish explicit live

Stable tag publish 이후 live release-integrity check를 실행해 npm latest, GitHub latest, GitHub Release installer asset이 모두 같은 version을 가리키는지 증명합니다.

완료 조건

  • GitHub Release의 install.sh hash가 checked-in installer와 일치.

App-Server Session Event Mapping

관측 이후 설계

완료 조건
  • Method shape가 versioned/bounded입니다.
  • desktop-app-server runtime surface는 관측 증거에서만 나옵니다.
  • CLI notify 부재를 Desktop 증거로 쓰지 않습니다.

Worktree App Instance Launcher

worktree evidence experimental

Codexus는 아직 git worktree별로 application instance 하나를 부팅하지 않습니다. Browser, log, app-health evidence를 변경별 instance에 묶기 전에 descriptor-backed launcher가 필요합니다.

완료 조건
  • cx app instance start/status/stop/logs --json가 owned instance artifact를 기록합니다.
  • 각 artifact는 worktree path, branch/head, command profile, pid/owner heartbeat, port 또는 URL, health check, log path, status를 포함합니다.
  • Port allocation은 worktree별이며, stop은 owned process만 정리하고 unrelated local server를 건드리지 않습니다.
  • Browser/dev-server evidence는 instance id를 참조하고 live evidence 없이 control/health를 주장하지 않습니다.

3. Gated / Blocked

가치는 있지만 upstream 또는 local contract가 생길 때까지 unavailable로 보여야 하는 기능입니다.

Live App-Server Turn Execution

blocked experimental

Desktop 관측과 session-event mapping이 별도로 증명되기 전까지 live turn execution을 켜지 않습니다.

Gate

지원되는 daemon 관측, non-disruptive attach, event mapping, 별도 enablement 설계.

Real Cron / Gateway Dispatch

blocked automation

현재 surface는 status와 dry-run audit record까지만 지원합니다.

Gate

Permission, approval, lock, dispatch, completion, recovery event가 필요합니다.

Statusline Integration

blocked upstream

Codex가 안정적인 statusline configuration surface를 제공할 때까지 cx session hud --json을 fallback으로 유지합니다.

Gate

문서화된 lifecycle/failure behavior를 가진 Codex 지원 설정면.

tmux-backed Worker Launch

blocked launch contract

Session state와 launch protocol이 안정될 때까지 session workers status는 gate report로 유지합니다.

Gate

Worker protocol, ownership model, cancellation behavior, evidence separation.

Native Subagent Active Launcher

blocked bridge 필요

Record/attach/complete/launch-contract surface는 유지하지만, Codexus가 native subagent를 spawn한다고 주장하지 않습니다.

Gate

지원되는 Codex bridge와 verification-only completion policy.

Automatic Context Injection

blocked approval 필요

Retrieved context는 reversible injection path가 설계되기 전까지 visible approval artifact로만 둡니다.

Gate

User-visible approval, audit artifact, disable path, no silent prompt mutation.
